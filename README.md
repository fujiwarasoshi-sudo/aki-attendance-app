# 別府店・日吉店 シフト・勤怠管理アプリ

スマートフォン／PCのブラウザで動くPWAです。共有Googleスプレッドシートの
シフトをSupabaseへ同期し、ログイン後の利用者だけが予定を閲覧できます。
公開ファイルには従業員名や実シフトを保存しません。

Supabaseを設定すると、従業員ログイン、クラウド打刻、管理者画面への
リアルタイム同期が有効になります。未設定時は従来の試作モードで動きます。

## 試せる機能

- 従業員・管理者画面の切り替え
- 出勤・退勤打刻
- 打刻時の位置情報取得
- 今後のシフト表示
- シフト希望、勤怠修正申請フォーム
- 管理者の出勤状況一覧と店舗絞り込み
- 店舗ごとの打刻可能範囲設定
- 2026年7月の日吉店シフト（6名）
- 従業員別・日別の予定表示
- 全従業員×31日の月間シフト一覧表
- 早番・遅番・休み・別府店・個別時間の色分け
- iPhone・Android向け下部メニュー
- ホーム画面へ追加できるPWA対応
- 通信が切れた場合の画面キャッシュ
- メールアドレス・パスワードによる従業員ログイン
- 出退勤・GPS・店舗判定のクラウド保存
- 管理者画面へのリアルタイム反映
- 従業員／管理者の閲覧権限制御
- 月次の実労働・残業時間集計
- 社労士提出用Excel（従業員別集計・日別明細）出力
- Googleスプレッドシートで読み込めるCSV出力
- 所定労働時間・標準休憩時間の変更
- シフト希望での有給休暇・半日有給申請
- 従業員別の有給付与日・付与日数・残日数管理
- 有給付与履歴を残す台帳方式
- 管理者による月間シフト作成・編集
- 早・遅・休み・別府・有給・個別時間のセル入力
- 前月シフトのコピーと月単位保存
- 保存した8月以降の予定を従業員画面へ共有
- Googleスプレッドシート更新時の自動同期
- 公開コードから従業員名・実シフトを除外

## シフト作成

- 管理画面の「シフトを作成」から対象月を選択する。
- スプレッドシートと同様に、行が従業員、列が日付の月間表で入力する。
- 各セルでは「早・遅・休み・別府・有給・個別時間・未設定」を選択できる。
- 個別時間は `9:00-14:00` のように入力する。
- 「前月をコピー」で前月の配置をたたき台にできる。
- 保存後はシフト一覧表と各従業員の予定へ反映される。

## 有給休暇管理

- 従業員は「希望を提出」から有給休暇を選び、1日または半日で申請する。
- 管理者は管理画面の「有給休暇管理」で従業員、付与日、付与日数を登録する。
- 残日数は `付与合計 - 承認済み使用日数` で表示する。
- 付与は上書きせず履歴として追加し、過去の付与根拠を確認できるようにする。
- 実際の付与日数、時効、繰越、時間単位年休等の扱いは就業規則と社労士確認後に確定する。

## クラウド共有版の設定

1. Supabaseで新しいプロジェクトを作成する。
2. SQL Editorで `supabase-schema.sql` を実行する。
3. Authentication の Users で従業員ユーザーを作成する。
   User Metadata に氏名と社員コードを登録する。
4. `config.js` に Project URL と anon public key を設定する。
5. SQL末尾の管理者設定例を使い、管理者の `role` を `admin` に変更する。
6. `stores` テーブルへ別府店・日吉店の緯度・経度、打刻可能半径を設定する。

`config.js` へ入れるのは anon public key だけです。
`service_role` key はアプリへ入れないでください。

## 共有版の動き

- 従業員が携帯からログインする。
- 出勤／退勤ボタンを押した時だけGPSを取得する。
- サーバー時刻、店舗、距離、GPS精度をクラウドへ保存する。
- 店舗外・測位不能・精度不足は「要確認」として保存する。
- 管理者画面は打刻データの変更を購読し、勤務中一覧を即時更新する。
- 従業員は自分の勤怠だけ、管理者は全従業員の勤怠を閲覧できる。
- 管理者は月を選び、実労働時間と残業時間を集計できる。
- Excelには「従業員別集計」と「日別勤怠明細」の2シートを出力する。
- 外部Excel変換ライブラリが利用できない場合も、Excel互換 `.xls` を生成する。

## 残業集計について

初期計算は次の式です。

`実労働時間 = 退勤時刻 - 出勤時刻 - 休憩時間`

現在の初期設定では、前残業は認めず、予定退勤後だけを残業として扱います。

`残業時間 = 実際の退勤時刻 - 予定退勤時刻（マイナスの場合は0）`

予定より早く出勤した時間は、実労働時間には記録しますが残業には加算しません。
管理画面の設定から、将来「実労働－所定時間」方式へ切り替えることもできます。

変形労働時間制、週単位の時間外労働、深夜・休日労働、時刻の丸め、
有給休暇等は別の判定が必要になる場合があります。本番運用前に社労士へ
計算ルールと提出列を確認し、設定を確定してください。

## 取込元

- 管理者が指定した共有Googleスプレッドシート
- シート名から対象年月を判定
- 氏名と社員コードの対応表は非公開のスクリプト設定に保存

## 起動方法

位置情報機能はセキュリティ上 `localhost` または HTTPS が必要です。

```sh
cd attendance-app
python3 -m http.server 8080
```

ブラウザで `http://localhost:8080` を開きます。

- 位置情報利用規程と従業員への説明
